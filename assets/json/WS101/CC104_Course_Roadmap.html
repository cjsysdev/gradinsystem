Foundations

Weeks 1–3 · Just enough C to reach arrays

Variables, decisions, loops, and functions — the minimum toolkit needed before arrays make sense.

Week 1 — C Foundations I

Sessions 1A–1B

Every big program is just small, precise instructions. What is a data structure vs. an algorithm?

1A

What are DS & Algorithms? First Hello World.

1B

Compiler/editor setup, reading error messages, first debugging.

Week 2 — C Foundations II

Sessions 2A–2B

Programs need to decide and repeat. How do we avoid retyping the same steps?

2A

if/else, switch — branching on a condition.

2B

for vs. while loops, accumulating a running total.

Week 3 — C Foundations III

Sessions 3A–3B

Don't repeat yourself. Name a block of steps once, reuse it anywhere.

3A

Defining & calling functions with parameters and return values.

3B

Consolidation: variables + loops + functions in one program.

Midterm

Weeks 4–9 · Arrays, start to finish

Six weeks building every array skill you'll need — storing, working through functions, finding, ordering, and gridding data — closing with a full review and the midterm exam.

Week 4 — Arrays I

Sessions 4A–4B

One box can hold one thing. What if we need a whole row of them?

4A

Declaration, indexing, pointers as addresses.

4B

Aggregation: sum, average, conditional count.

Week 5 — Functions + Arrays

Sessions 5A–5B

Week 3 built functions with single values. Now a function needs to work with a whole row of lockers.

5A

How to create a function, revisited: multiple parameters, return types, functions calling other functions — the shape every array function will follow.

5B

How to use an array in a function: passing arr[] + size, why the original array changes when the function modifies it, rewriting Week 4's sum/average as standalone functions.

Week 6 — Arrays II: Searching Variations

Sessions 6A–6B

Data is stored. Now how do we find something in it — as a function?

6A

Linear search — the basic loop-and-compare pattern, built as its own function.

6B

Variations: first match, all matches, min/max.

Week 7 — Arrays III: Sorting Variations

Sessions 7A–7B

An organized line is easier to search. How do we put data in order ourselves?

7A

Bubble sort — by hand, then in code.

7B

Descending order, sort + search combined.

Week 8 — 2D Arrays / Matrices

Sessions 8A–8B

What if our row of lockers needs its own rows? Representing a grid.

8A

grid[row][col], row-major layout, grid traversal.

8B

Matrix addition, grid search, diagonal sum.

Week 9 — Beyond Arrays + Review

Sessions 9A–9B · MIDTERM EXAM

Recognize the rest of the toolbox by name. Then prove what you've built.

9A

Linked lists, stacks & queues — theory only, no code.

9B

Full array review, midterm exam. Submit Concept Brief — the record your CRUD project will manage.

Finals

Weeks 10–17 · Your CRUD project, one piece per session

Starting Week 10, almost every session ends with a worksheet that adds one real, working piece to the same arrays-only CRUD program — no separate "project week" at the end. Everything stays arrays: parallel arrays for records, no structs, no pointers beyond what Weeks 4–5 already covered.

CreateReadUpdateDeleteSearchFilterSortFast LookupRecursive FeatureMenu / Integration

Week 10 — CRUD I: Create & Read

Sessions 10A–10B

Your Concept Brief becomes real code. Design the record, then get data in and out.

10A

Design the record as parallel arrays (e.g. names[], scores[], plus a count) sized to a max capacity.

Worksheet → addRecord()
10B

Build and test a function that lists every current record, formatted and aligned.

Worksheet → displayAll()

Week 11 — CRUD II: Update & Delete

Sessions 11A–11B

Finding a record is one thing. Now change it — or take it out cleanly.

11A

Locate a record by field, then edit the matching slot(s) across the parallel arrays.

Worksheet → updateRecord()
11B

Remove a record and shift the remaining elements left to close the gap; decrement count.

Worksheet → deleteRecord()

Week 12 — CRUD III: Search Your Records

Sessions 12A–12B

Week 6's search patterns, pointed at your own data. Find one record — or every record that fits.

12A

Reuse the linear-search pattern to find a record by name or ID and return its index.

Worksheet → searchByField()
12B

Reuse the find-all pattern to list every record matching a condition (e.g. score above X).

Worksheet → filterRecords()

Week 13 — CRUD IV: Sort Your Records

Sessions 13A–13B

Bubble sort gets a name for how fast it is. Then it goes to work ordering your list.

13A

Big-O, formalized: count comparisons/swaps in your own bubble sort code from Week 7.

Worksheet → measure it
13B

Sort the record list by a chosen field, ascending or descending, keeping parallel arrays in sync.

Worksheet → sortRecords()

Week 14 — CRUD V: Fast Lookup

Sessions 14A–14B

A sorted list unlocks a faster search. Binary search, on your own data.

14A

Divide-and-conquer logic: check the middle, discard half, repeat — traced by hand first.

Lecture → binary search logic
14B

Add binary search as a "fast lookup" option on the now-sorted record list; compare comparison counts to linear search.

Worksheet → fastLookup()

Week 15 — CRUD VI: One Recursive Feature

Sessions 15A–15B

One new idea: a function that calls itself. Then it earns a real spot in the project.

15A

Base case + recursive case, practiced on familiar array tasks (recursive sum, recursive count).

Lecture → recursion basics
15B

Add one recursive function to the project (e.g. recursive count of matching records, or recursive display).

Worksheet → recursive feature

Week 16 — Beyond Arrays (Theory Only)

Sessions 16A–16B

Meet the rest of the toolbox by name. Then spend the lab tying your project into one program.

16A

Linked lists, stacks (LIFO) & queues (FIFO) — analogies and real-world examples, no code, not tested.

16B

Trees & graphs, by name only. Lab time: wire every piece so far into one menu-driven main() loop.

Worksheet → menu integration

Week 17 — Final Review + FINAL EXAM

Sessions 17A–17B · FINALS

Everything since Week 1, in one working program. Show it, then defend it.

17A

Full-course review: arrays, sorting, searching, recursion — code-tracing drills.

17B

Final exam, plus presentation/defense of the completed CRUD project.
