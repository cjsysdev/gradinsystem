CSS Cascading and Selector Priority

Interactive Guessing Game – Which Style Wins?

Instructions

Read each CSS snippet carefully. Guess which color or style will apply, then click "Check Answer" to reveal the correct result.

ID vs Class vs Element

<p class="note" id="message">Hello, world!</p>
p { color: green; }
.note { color: blue; }
#message { color: red; }
Answer: Red – ID selector has the highest specificity.

Source Order

<p class="info">Welcome to CSS!</p>
.info { color: purple; }
.info { color: orange; }
Answer: Orange – The last rule wins if specificity is the same.

Inline vs External

<p id="title" style="color: brown;">Inline Text</p>
#title { color: black; }
Answer: Brown – Inline styles override external styles.

Using !important

<p id="note" class="highlight">Important text</p>
.highlight { color: green !important; }
#note { color: red; }
Answer: Green – !important overrides the ID rule.

Multiple Classes

<p class="info note">Hello Cascading</p>
.info { color: blue; }
.note { color: orange; }
Answer: Orange – The last declared class rule wins if specificity is equal.

Parent vs Child Selector

<div class="wrapper">
  <p>Styled Text</p>
</div>
.wrapper p { color: teal; }
p { color: gray; }
Answer: Teal – .wrapper p is more specific than p.

Attribute Selector

<input type="text" value="hello">
input { border: 2px solid black; }
input[type="text"] { border: 2px solid red; }
Answer: Red border – Attribute selector is more specific than element selector.

Pseudo-class

<a href="#" class="link">Hover me</a>
.link { color: blue; }
.link:hover { color: orange; }
Answer: Orange when hovered – :hover applies dynamically.

Inline vs !important

<p style="color: gray;" class="warn">Warning</p>
.warn { color: red !important; }
Answer: Red – !important even overrides inline styles.

Combining IDs and Classes

<p id="msg" class="note">Final Test</p>
#msg.note { color: blue; }
#msg { color: green; }
Answer: Blue – #msg.note is more specific (110 vs 100).

Pseudo-class vs. Class Selector

<a href="#" class="link">Click Me</a>
.link { color: green; }
a:hover { color: red; }
Answer: Red – a:hover applies when hovered.

Combining Class and Element Selectors

<p class="highlight">CSS Practice</p>
p { color: black; }
p.highlight { color: blue; }
Answer: Blue – p.highlight is more specific (11 vs 1).

Multiple Classes

<p class="note important">Read Me!</p>
.note { color: gray; }
.important { color: orange; }
Answer: Orange – The .important rule appears later and overrides.

Universal Selector

<h2>Title</h2>
* { color: purple; }
h2 { color: green; }
Answer: Green – h2 is more specific than *.

Attribute Selector

<input type="text" value="Hello">
input { color: black; }
input[type="text"] { color: red; }
Answer: Red – The attribute selector is more specific (11 vs 1).

Multiple Selectors with Same Specificity

<h1 class="main">Hello</h1>
.main { color: green; }
h1.main { color: blue; }
Answer: Blue – h1.main is more specific (11 vs 10).

ID and Inline Combination

<h2 id="subtitle" style="color:pink;">Subheading</h2>
#subtitle { color: green; }
Answer: Pink – Inline styles (1000) override ID selectors (100).

Pseudo-element vs. Class

<p class="intro">Welcome!</p>
.intro::first-letter { color: red; }
.intro { color: blue; }
Answer: Red – The ::first-letter targets the letter specifically.

!important and Inline Style

<p style="color:yellow;" class="alert">Warning!</p>
.alert { color: red !important; }
Answer: Red – !important overrides inline styles.

Nested Selectors

<div class="container">
  <p>Nested Text</p>
</div>
.container p { color: blue; }
p { color: gray; }
Answer: Blue – .container p is more specific (11 vs 1).
